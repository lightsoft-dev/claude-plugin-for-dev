# 개발 워크플로우 — E2E 연동 규칙

## 기능 변경 시 E2E 대응

| 상황 | E2E 대응 | 강도 |
|------|---------|------|
| 새 기능 추가 | E2E 작성 + 테스트 | **필수** |
| 기존 기능 로직 수정 | 관련 E2E 실행 (깨지면 수정) | **필수** |
| UI/공통 컴포넌트 변경 | 영향 범위 파악 후 사용자 컨펌 받고 진행 | **컨펌 후 진행** |

### 세부 규칙

1. **새 기능 추가**: 기능 구현 완료 후 E2E 테스트를 작성하고 실행한다
2. **기존 기능 로직 수정**: 해당 도메인의 E2E 테스트를 실행하여 깨지는 것이 없는지 확인한다. 깨지면 수정한다
3. **UI/공통 컴포넌트 변경**: 영향 범위가 넓을 수 있으므로, 어떤 E2E를 돌릴지 사용자에게 컨펌을 받고 진행한다
