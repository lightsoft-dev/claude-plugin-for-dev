# E2E 테스트 보고서 규칙

## 파일 위치 및 네이밍

```
docs/e2e-reports/{날짜}_{목적}.md        ← skip 남아있음 (작업 중)
docs/e2e-reports/{날짜}_{목적}_완료.md    ← 전부 해결됨
```

예: `2026-02-22_fair-사전준비-계약.md` → 전부 해결 시 → `2026-02-22_fair-사전준비-계약_완료.md`

## 보고서 내용

```markdown
# {날짜} {목적} E2E

## 테스트 범위
- 테스트한 페이지/도메인 목록

## 결과
- 통과: N개
- 실패 (E2E 문제 → 수정완료): N개
- 실패 (로직 문제 → skip): N개

## skip된 테스트 목록
| 파일 | 테스트명 | 문제 | 비고 |
|------|---------|------|------|
| ... | ... | ... | ... |

## 수정 이력
| 날짜 | # | 수정 내용 | 결과 |
|------|---|----------|------|
| ... | ... | ... | 통과 / 또 실패 |
```

## 필수 규칙

- `test.skip()` 처리 시 **반드시 보고서에 기록** (skip과 문서화는 세트)
- skip된 테스트 원본 테이블은 수정하지 않는다 (수정 이력 섹션에만 추가)
- 전체 skip이 해결되면 파일명에 `_완료` 붙이기
- 명시적 지정 없으면 **가장 최근 보고서**를 대상으로 작업
