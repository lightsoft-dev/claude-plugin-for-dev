# E2E Fix

$ARGUMENTS

## 실행 단계

1. `docs/e2e-reports/`에서 최신 보고서 읽기 (명시적 지정 없으면 가장 최근 파일)
2. skip된 테스트 목록에서 수정 대상 코드 파악
3. 프론트(`page.tsx`) + 백엔드(`actions/*.ts`) 코드 읽고 로직 수정
4. 해당 테스트의 `test.skip()` 해제
5. 그 테스트만 실행 (`npx playwright test 파일명.spec.ts`)
6. 결과에 따라:
   - 통과 → 보고서 수정 이력에 기록
   - 실패 → 재분류 (E2E 문제 vs 로직 문제) 후 반복
7. 전체 skip 해결 시 파일명에 `_완료` 붙이기

## 주의

Server Action이나 공통 컴포넌트를 수정한 경우, 이미 통과했던 다른 테스트에도 영향을 줄 수 있다. 수정 범위가 클 때는 관련 테스트가 깨지지 않았는지 주의할 것.
